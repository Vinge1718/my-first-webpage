READMEs and how to go about formatting them


let's start with a list of what Moringa School recommends ones README (in markdown syntax for lists (both types))

  1. 1. Application name written as a head-ing/er (===)/(#)
  2. 2. Names of contributors written as a header and do the other section heads (---)/(##)
  3. 3. links syntax include ([link description](link-url)) or ([linkdescription](link-url)) urls in
    angle brackets will be turned into links
  4. 3. Description of the projects purpose
  5. 4. Complete set up and instalation guide
  6. 5. Licence information with a copyright and date
  7. 6. Line identations are done by use of (...) and spacing between paragraphs is done by double spacing
  8. 7. Bolding and italics both use (asterics ** or Underscores _) respectively
  9. 8. Unordered lists can be done through (+) (-) (*)

More information to consider adding includes

  • Technologies added
  • known bugs
  • contact information
  • Suppoer and contribution instructions
